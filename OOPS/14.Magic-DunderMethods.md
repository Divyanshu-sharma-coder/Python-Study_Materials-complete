# Magic / Dunder Methods in Python

In Python's Object-Oriented Programming (OOP) syntax, **Magic Methods** (also widely referred to as **Dunder Methods** due to their Double Underscore prefixes and suffixes) are special pre-defined methods that allow you to emulate built-in type operations for your custom classes.

You rarely call dunder methods directly like `obj.__init__()`. Instead, they run automatically behind the scenes when you execute core native operations, such as calling `len(obj)`, performing arithmetic operations like `x + y`, or printing an object directly to the terminal.

---

# 1. What Are Magic Methods?

Magic methods are special methods that begin and end with double underscores:

```python
__init__
__str__
__repr__
__len__
__call__
```

Python automatically invokes them whenever certain built-in operations are performed on an object.

These methods allow your custom classes to behave like built-in Python data types.

---

# 2. The Constructor: `__init__`

The most commonly used magic method is `__init__`.

It acts as a constructor and executes automatically whenever a new object is instantiated.

---

## Example

```python
class Employee:
    def __init__(self, name):
        self.name = name

emp = Employee("Harry")
```

### What Happens Internally?

```python
emp = Employee("Harry")
```

Python automatically executes:

```python
Employee.__init__(emp, "Harry")
```

---

# 3. Overloading Length with `__len__`

Normally, Python's `len()` function works only on built-in collections like strings, lists, tuples, and dictionaries.

Custom objects don't support `len()` unless you define `__len__()`.

---

## Example

```python
class Employee:
    def __init__(self, name):
        self.name = name

    def __len__(self):
        return len(self.name)

emp = Employee("Harry")

print(len(emp))
```

### Output

```text
5
```

### Behind the Scenes

```python
len(emp)
```

becomes:

```python
emp.__len__()
```

---

# 4. String Representation Methods

When printing an object directly, Python displays a default memory reference:

```python
<__main__.Employee object at 0x104bfa4d0>
```

To customize this behavior, Python provides:

- `__str__()`
- `__repr__()`

---

# 5. The `__str__()` Method

The `__str__()` method returns a human-friendly string representation.

It is automatically called when using:

```python
print(obj)
str(obj)
```

---

## Example

```python
class Employee:
    def __init__(self, name):
        self.name = name

    def __str__(self):
        return f"Employee Standard Profile: {self.name}"

emp = Employee("Harry")

print(emp)
```

### Output

```text
Employee Standard Profile: Harry
```

---

# 6. The `__repr__()` Method

The `__repr__()` method is intended for developers.

It should provide a detailed and ideally recreatable representation of the object.

---

## Example

```python
class Employee:
    def __init__(self, name):
        self.name = name

    def __repr__(self):
        return f"Employee('{self.name}')"

emp = Employee("Harry")

print(repr(emp))
```

### Output

```text
Employee('Harry')
```

---

# 7. `__str__()` vs `__repr__()`

## Example

```python
class Employee:
    def __init__(self, name):
        self.name = name

    def __str__(self):
        return f"Employee Standard Profile: {self.name}"

    def __repr__(self):
        return f"Employee('{self.name}')"

emp = Employee("Harry")

print(str(emp))
print(repr(emp))
```

### Output

```text
Employee Standard Profile: Harry
Employee('Harry')
```

---

## Fallback Rule

If `__str__()` is not defined:

```python
print(obj)
```

automatically falls back to:

```python
obj.__repr__()
```

---

# 8. Making Objects Callable with `__call__`

The `__call__()` method allows an object instance to behave like a normal function.

Once implemented, you can execute the object using parentheses.

---

## Example

```python
class CallableTool:
    def __call__(self):
        print("The object instance was executed exactly like a function!")

tool = CallableTool()

tool()
```

### Output

```text
The object instance was executed exactly like a function!
```

---

## Behind the Scenes

```python
tool()
```

becomes:

```python
tool.__call__()
```

---

# 9. Real-World Examples of Magic Methods

### Arithmetic Operator Overloading

```python
class Number:
    def __init__(self, value):
        self.value = value

    def __add__(self, other):
        return Number(self.value + other.value)

n1 = Number(5)
n2 = Number(10)

result = n1 + n2

print(result.value)
```

### Output

```text
15
```

Python converts:

```python
n1 + n2
```

into:

```python
n1.__add__(n2)
```

---

# 10. Why Are Magic Methods Important?

Magic methods allow custom classes to integrate naturally with Python's built-in syntax.

Without them:

```python
print(obj)
len(obj)
obj()
obj1 + obj2
```

would fail.

With magic methods, your classes behave like native Python objects.

---

# Summary Reference Table

| Built-In Python Action | Triggered Dunder Method | Purpose |
|-----------------------|------------------------|----------|
| `Object()` | `__init__()` | Initializes object state |
| `len(object)` | `__len__()` | Returns object length |
| `print(object)` | `__str__()` | User-friendly representation |
| `repr(object)` | `__repr__()` | Developer/debug representation |
| `object()` | `__call__()` | Makes objects callable |
| `obj1 + obj2` | `__add__()` | Addition operator overloading |
| `obj1 == obj2` | `__eq__()` | Equality comparison |
| `obj[key]` | `__getitem__()` | Indexing behavior |
| `obj[key] = value` | `__setitem__()` | Assignment through indexing |

---

# Key Takeaways

- Magic Methods (Dunder Methods) are special Python methods surrounded by double underscores.
- Python calls them automatically when specific operations occur.
- `__init__()` initializes objects.
- `__len__()` customizes the behavior of `len()`.
- `__str__()` controls user-facing text output.
- `__repr__()` controls developer/debug output.
- `__call__()` allows objects to behave like functions.
- Operator overloading is powered by dunder methods like `__add__()` and `__eq__()`.
- They make custom classes feel and behave like native Python types.

> Magic Methods are the bridge between your custom classes and Python's built-in language features, enabling objects to participate naturally in printing, arithmetic, comparisons, indexing, iteration, and much more.
