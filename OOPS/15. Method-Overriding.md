# Method Overriding in Python

## 1. What is Method Overriding?

Method overriding is a powerful feature of Object-Oriented Programming (OOP) that allows a child class (subclass) to redefine or change a method that it has already inherited from its parent class (superclass).

This is highly useful when the behavior or logic provided by the parent class's method isn't specific or accurate enough for the specialized child class.

---

## 2. Practical Code Implementation

Below is the precise implementation demonstrated in the video, featuring a base `Shape` class, a generic rectangle area calculation, and a specialized `Circle` class that overrides the logic using the `super()` keyword:

```python
class Shape:
    def __init__(self, x, y):
        self.x = x
        self.y = y
        
    def area(self):
        return self.x * self.y

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
        super().__init__(radius, radius)
        
    def area(self):
        return 3.14 * super().area()

# Instantiating a generic Shape (Rectangle)
rec = Shape(3, 5)
print(rec.area())

# Instantiating a Circle subclass
c = Circle(5)
print(c.area())
```

---

## 3. Code Execution Flow Explanation

### Base Rectangle Flow

When you instantiate:

```python
rec = Shape(3, 5)
```

and call:

```python
rec.area()
```

it directly executes the logic inside the `Shape` class, multiplying:

```text
3 × 5 = 15
```

and returns:

```text
15
```

---

### Circle Overriding Flow

When:

```python
c = Circle(5)
```

is initialized:

1. The `Circle.__init__()` constructor runs.

2. It uses:

```python
super().__init__(radius, radius)
```

to call the parent `Shape` constructor.

3. Both `self.x` and `self.y` become:

```text
5
```

4. When:

```python
c.area()
```

is called, Python first checks the `Circle` class.

5. Since `Circle` contains its own `area()` method, the parent's version is overridden.

6. Inside:

```python
super().area()
```

Python moves up to the parent class implementation:

```python
def area(self):
    return self.x * self.y
```

which calculates:

```text
5 × 5 = 25
```

7. The overridden method then computes:

$$
3.14 \times 25
$$

which equals:

$$
78.5
$$

## 6. Key Takeaways

### `super()` Keyword

- Acts as a helper tool to execute parent class methods inside child class methods.
- Helps avoid code duplication.
- Maintains clean inheritance structures.
- Makes constructor chaining easier.

### Method Overriding Benefits

- Allows specialized behavior in child classes.
- Maintains a common interface across related classes.
- Improves code readability and scalability.
- Supports polymorphism in Object-Oriented Programming.

### Real-World Example

A generic `Shape` can calculate area using:

```text
length × width
```

But a `Circle` requires:

$$
\pi r^2
$$

Instead of creating a completely different method name like:

```python
circle_area()
```

we override the existing:

```python
area()
```

method, keeping a consistent interface across all shapes.

---

## Final Definition

> **Method Overriding** is an Object-Oriented Programming feature in which a subclass provides its own implementation of a method that already exists in its parent class. The overridden method in the child class is executed instead of the parent version whenever it is called through an object of the child class.
