# Multiple Inheritance in Python

## 1. What is Multiple Inheritance?

Multiple inheritance is a feature in Object-Oriented Programming (OOP) where a single **child class (subclass/derived class)** inherits attributes and methods from **more than one parent class (base classes)**.

---

## Analogy

Think of a person who is both a **corporate employee** and a **dancer**.

So in code:

- `Employee` → gives job-related behavior  
- `Dancer` → gives dance-related behavior  
- `DancerEmployee` → inherits both

---

## Structure Diagram

```
        ┌──────────────────┐       ┌──────────────────┐
        │  Parent Class 1  │       │  Parent Class 2  │
        │   (Employee)     │       │    (Dancer)      │
        └────────┬─────────┘       └────────┬─────────┘
                 │                          │
                 └──────────┬──────────────┘
                            │
                            ▼
              ┌────────────────────────┐
              │     Child Class        │
              │  (DancerEmployee)      │
              └────────────────────────┘
```

---

## 2. Practical Code Implementation

```python
class Employee:
    def __init__(self, name):
        self.name = name

    def show(self):
        print(f"The name is {self.name}")


class Dancer:
    def __init__(self, dance):
        self.dance = dance

    def show(self):
        print(f"The dance is {self.dance}")


class DancerEmployee(Employee, Dancer):
    def __init__(self, name, dance):
        self.name = name
        self.dance = dance


# Creating object
o = DancerEmployee("Shivani", "Kathak")

print(o.name)
print(o.dance)

o.show()
```

---

## 3. Method Conflict Problem

Both parent classes have:

```python
show()
```

So Python must decide which one to use.

---

## 4. Method Resolution Order (MRO)

Python resolves method calls using **left-to-right priority order** in the inheritance list.

### Case 1:

```python
class DancerEmployee(Employee, Dancer):
```

Then:

```text
Employee.show() is used
```

Output:
```
The name is Shivani
```

---

### Case 2:

```python
class DancerEmployee(Dancer, Employee):
```

Then:

```text
Dancer.show() is used
```

Output:
```
The dance is Kathak
```

---

## 5. Checking MRO

```python
print(DancerEmployee.mro())
```

---

## 6. MRO Output Explanation

```
[<class '__main__.DancerEmployee'>,
 <class '__main__.Employee'>,
 <class '__main__.Dancer'>,
 <class 'object'>]
```

### Meaning:

Python searches in this order:

```
DancerEmployee → Employee → Dancer → object
```

It stops at the first match.

---

## 7. Key Takeaways

✔ Multiple inheritance = one child → multiple parents  
✔ Method name conflict resolved using **MRO**  
✔ Left-to-right order matters  
✔ `.mro()` shows method lookup path  

---

## Final Idea

Multiple inheritance lets a class combine features of multiple classes, but Python controls ambiguity using **Method Resolution Order (MRO)**.
