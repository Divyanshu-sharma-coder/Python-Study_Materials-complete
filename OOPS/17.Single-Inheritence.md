# Single Inheritance in Python

## 1. What is Single Inheritance?

Single inheritance is the simplest form of inheritance in Object-Oriented Programming (OOP). It occurs when a single **child class (subclass/derived class)** inherits from a single **parent class (superclass/base class)**.

### Structure Diagram

```
┌────────────────────┐
│   Parent Class     │
│   (Base Class)     │
└─────────┬──────────┘
          │
          ▼
┌────────────────────┐
│   Child Class      │
│ (Derived Class)    │
└────────────────────┘
```

---

## Key Benefits

✔ **Code Reusability**  
- Child class automatically gets parent class features

✔ **Easy Maintenance**  
- Changes in parent class reflect in child class

---

## 2. Practical Code Implementation

```python
class Animal:
    def __init__(self, name, species):
        self.name = name
        self.species = species

    def make_sound(self):
        print("Sound made by the animal")


class Dog(Animal):
    def __init__(self, name, breed):
        # Calling parent class constructor
        Animal.__init__(self, name, species="Dog")
        self.breed = breed

    def make_sound(self):
        print("Bark!")


# Creating objects
d = Dog("Tommy", "Labrador")
d.make_sound()

a = Animal("Generic Animal", "Unknown")
a.make_sound()
```

---

## 3. Code Breakdown & Flow

### ✔ Object Creation Flow

When you run:

```python
d = Dog("Tommy", "Labrador")
```

Python executes:

- `Dog.__init__()`
- Calls `Animal.__init__()`
- Sets:
  ```
  name = Tommy
  species = Dog
  breed = Labrador
  ```

---

### ✔ Method Overriding Flow

Both classes have:

```python
make_sound()
```

So:

- `d.make_sound()` → uses **Dog class**
- `a.make_sound()` → uses **Animal class**
