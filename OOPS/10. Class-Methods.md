# Class Methods in Python (`@classmethod`)

In Python, regular methods usually operate on individual objects and receive `self` as their first argument.

However, sometimes you need a method that works directly with the **Class itself** rather than any specific object.

For these situations, Python provides the **`@classmethod`** decorator.

A class method receives the class (`cls`) as its first argument instead of an object instance (`self`).

---

# 1. Why Do We Need Class Methods?

Suppose you have a shared class variable:

```python
class Employee:

    company = "Apple"

    def __init__(self, name):
        self.name = name
```

The variable:

```python
company
```

belongs to the class, not to individual objects.

---

# The Problem

Imagine trying to change the company using a regular instance method.

```python
class Employee:

    company = "Apple"

    def __init__(self, name):
        self.name = name

    def change_company_inefficiently(self, new_company):

        self.company = new_company
```

---

# Creating an Object

```python
e1 = Employee("Harry")
```

---

# Modifying Company

```python
e1.change_company_inefficiently("Tesla")
```

---

# Checking Results

```python
print(e1.company)
```

Output:

```text
Tesla
```

---

```python
print(Employee.company)
```

Output:

```text
Apple
```

---

# What Happened?

Many beginners assume:

```python
self.company = new_company
```

modifies the class variable.

It does not.

Python creates a brand-new instance variable:

```python
e1.company
```

which shadows:

```python
Employee.company
```

The class variable remains unchanged.

---

# The Solution: Class Methods

Instead of receiving:

```python
self
```

we receive:

```python
cls
```

which points directly to the class blueprint.

---

# 2. Syntax of Class Methods

Use the decorator:

```python
@classmethod
```

above the method definition.

---

## Basic Structure

```python
class Employee:

    company = "Apple"

    @classmethod
    def change_company(cls, new_company):

        cls.company = new_company
```

---

# Understanding cls

Think of:

```python
cls
```

as:

```python
Employee
```

being passed automatically.

---

When Python executes:

```python
Employee.change_company("Tesla")
```

it internally behaves like:

```python
Employee.change_company(Employee, "Tesla")
```

---

# 3. Complete Example

```python
class Employee:

    company = "Apple"

    def __init__(self, name):

        self.name = name

    def show(self):

        print(
            f"Employee: {self.name}"
        )

        print(
            f"Company: {self.company}"
        )

    @classmethod
    def change_company(cls, new_company):

        cls.company = new_company
```

---

# Creating an Object

```python
e1 = Employee("Harry")
```

---

# Before Modification

```python
e1.show()
```

Output:

```text

Company: Apple
```

---

# Modifying Company

```python
e1.change_company("Tesla")
```

---

# After Modification

```python
e1.show()
```

Output:

```text
Employee: Harry
Company: Tesla
```

---

# Checking the Class Variable

```python
print(Employee.company)
```

Output:

```text
Tesla
```

Now the actual class variable has changed.

# 4. Calling Class Methods

A class method can be called in two ways.

---

## Through the Class

```python
Employee.change_company("Tesla")
```

Recommended approach.

---

## Through an Object

```python
e1.change_company("Tesla")
```

Also valid.

---

Both produce the same result because Python still passes:

```python
Employee
```

to `cls`.

---

# Example

```python
class Employee:

    company = "Apple"

    @classmethod
    def change_company(cls, company):

        cls.company = company


Employee.change_company("Tesla")
```

---

```python
print(Employee.company)
```

Output:

```text
Tesla
```

# 5. Instance Method vs Class Method

Consider:

```python
class Employee:

    company = "Apple"

    def instance_method(self):

        print(self.name)

    @classmethod
    def class_method(cls):

        print(cls.company)
```

---

## Instance Method

Receives:

```python
self
```

Works with:

```python
Object Data
```

---

## Class Method

Receives:

```python

```

Works with:

```python
Class Data
```

# 6. Alternative Constructors

One of the most important uses of class methods is creating alternative constructors.

---

# The Problem

Suppose employee data arrives like:

```text
Harry-1200
```

Instead of manually splitting every string:

```python
name, salary = data.split("-")
```

we can create a class method that automatically builds objects.

---

# Example

```python
class Employee:

    def __init__(self, name, salary):

        self.name = name
        self.salary = salary

    @classmethod
    def from_string(cls, data):

        name, salary = data.split("-")

        return cls(name, int(salary))
```

---

# Creating Object

```python
emp = Employee.from_string(
    "Harry-1200"
)
```

---

Equivalent to:

```python
emp = Employee(
    "Harry",
    1200
)
```

---

# Verification

```python
print(emp.name)
```

Output:

```text
Harry
```

---

```python
print(emp.salary)
```

Output:

```text
1200
```

---

# Why Use cls Instead of Employee?

Notice:

```python
return cls(name, salary)
```

instead of:

```python
return Employee(name, salary)
```

This makes inheritance work correctly.

---

# Example

```python
class Programmer(Employee):
    pass
```

Now:

```python
Programmer.from_string(
    "Rohan-5000"
)
```

returns:

```python
Programmer Object
```

not an Employee object.

---

# 7. When Should You Use Class Methods?

Use class methods when:

---

### Modifying Shared Class Variables

```python
cls.company = "Tesla"
```

---

### Building Alternative Constructors

```python
Employee.from_string(...)
```

---

### Managing Class-Level Configuration

```python
cls.tax_rate = 0.18
```

---

### Creating Factory Methods

```python
Car.from_json(...)
```

```python
User.from_database(...)
```

```python
Employee.from_csv(...)
```

---

# 8. Instance Methods vs Class Methods vs Static Methods

| Feature | Instance Method | Class Method | Static Method |
|----------|---------------|--------------|---------------|
| Decorator | None | `@classmethod` | `@staticmethod` |
| First Argument | `self` | `cls` | None |
| Access Instance Variables | ✅ Yes | ❌ No | ❌ No |
| Access Class Variables | ✅ Yes | ✅ Yes | ❌ No |
| Modify Class Variables | ⚠️ Indirectly | ✅ Yes | ❌ No |
| Common Usage | Object Operations | Class Operations | Utility Functions |

# Key Takeaways

### Instance Method

```python
def method(self):
```

✅ Works with individual objects

✅ Accesses instance variables

---

### Class Method

```python
@classmethod

def method(cls):
```

✅ Works with the class itself

✅ Can modify class variables

✅ Useful for alternative constructors

---

### Static Method

```python
@staticmethod

def method():
```

✅ Utility/helper functions

✅ No access to instance or class state

---

### Most Important Rule

When you need to work with:

```python
Employee.company
```

or create alternative constructors like:

```python
Employee.from_string(...)
```

use:

```python
@classmethod
```

because it gives direct access to the class through the `cls` parameter.
