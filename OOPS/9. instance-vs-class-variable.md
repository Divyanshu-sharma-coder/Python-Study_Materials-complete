# Instance Variables vs Class Variables in Python

In Object-Oriented Programming (OOP), data can exist at two different levels:

1. **Instance Level** тЖТ Data belongs to a specific object.
2. **Class Level** тЖТ Data belongs to the class itself and is shared among all objects.

Understanding the difference between Instance Variables and Class Variables is essential because it directly affects memory allocation, object behavior, and data sharing across your program.

---

# 1. What are Instance Variables?

Instance Variables are variables that belong to a specific object instance.

Every object receives its own independent copy of these variables.

Changes made to one object's instance variables do not affect any other object.

---

## Characteristics

тЬЕ Unique for every object

тЬЕ Usually defined inside `__init__()`

тЬЕ Accessed through `self`

тЬЕ Stored separately in memory for each object

---

## Syntax

```python
class Employee:

    def __init__(self, name):

        self.name = name
        self.raise_amount = 0.02
```

---

## Example

```python
class Employee:

    def __init__(self, name):

        self.name = name
        self.raise_amount = 0.02


emp1 = Employee("Harry")
emp2 = Employee("Rohan")
```

---

### Memory Representation

```text

```

---

```text

raise_amount = 0.02
```

---

Although both objects contain:

```python
raise_amount
```

they are separate memory locations.

---

# Modifying an Instance Variable

```python

```

---

### Result


# 2. What are Class Variables?

Class Variables belong to the class itself.

Instead of creating a new copy for every object, Python creates one shared copy.

All objects reference that same variable.

---

## Characteristics

тЬЕ Shared among all objects

тЬЕ Defined outside methods

тЬЕ Stored once in memory

тЬЕ Useful for common configurations

---

## Syntax

```python
class Employee:

    company_name = "Apple"
```

---

## Example

```python
class Employee:

    company_name = "Apple"

    def __init__(self, name):

        self.name = name
```

---

### Creating Objects

```python
emp1 = Employee("Harry")
emp2 = Employee("Rohan")
```

---

### Accessing Class Variable

```python
print(emp1.company_name)
print(emp2.company_name)
```

Output:

```text
Apple
Apple
```

# 8. Complete Example

```python
class Employee:

    company_name = "Apple"

    no_of_employees = 0

    def __init__(self, name):

        self.name = name

        self.raise_amount = 0.02

        Employee.no_of_employees += 1

    def show_details(self):

        print(
            f"Employee: {self.name}"
        )

        print(
            f"Raise: {self.raise_amount}"
        )

        print(
            f"Company: {self.company_name}"
        )


emp1 = Employee("Harry")
emp2 = Employee("Rohan")


emp1.company_name = "Apple India"

emp1.raise_amount = 0.30


emp1.show_details()

emp2.show_details()


print(
    Employee.no_of_employees
)
```

---

### Output

```text
Employee: Harry
Raise: 0.3
Company: Apple India

Employee: Rohan
Raise: 0.02
Company: Apple

2
```

# 9. Instance Variables vs Class Variables

| Feature | Instance Variables | Class Variables |
|----------|-------------------|----------------|
| Defined In | Methods (`__init__`) | Class Body |
| Accessed Through | `self.variable` | `ClassName.variable` |
| Memory Allocation | Separate copy per object | Single shared copy |
| Modification Impact | Affects one object | Affects all objects |
| Ownership | Object | Class |
| Typical Use | Names, IDs, User Data | Configurations, Counters |

# Key Takeaways

### Instance Variables

```python
self.variable
```

тЬЕ Unique for every object

тЬЕ Stored separately

тЬЕ Used for object-specific data

---

### Class Variables

```python
variable = value
```

inside the class body.

тЬЕ Shared among all objects

тЬЕ Stored once

тЬЕ Used for common settings and counters

---

### Lookup Rule

Python searches:

```text
Object
   тЖУ
Class
   тЖУ
Error
```

---

### Most Important Rule

Assigning through an object:

```python
emp1.company_name = "Apple India"
```

does **not modify the class variable**.

Instead, Python creates a new instance variable that shadows the class variable for that object only.
