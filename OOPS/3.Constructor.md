# Constructors in Python: `__init__` and Object Initialization

When creating objects from a class template, you often need to assign unique values immediately.

Without constructors, developers would need to manually assign attributes after creating every object, which becomes inefficient and error-prone as applications grow.

A **Constructor** is a special method that executes automatically whenever a new object is created.

Its primary purpose is to initialize the object's state.

---

# 1. What is a Constructor?

A **Constructor** is a special method that runs automatically when an object is instantiated.

In Python, constructors are implemented using the reserved method:

```python
__init__()
```

This method is known as a **dunder method** (Double Underscore Method).

---

## Why Constructors Exist

Without a constructor:

```python
class Person:
    pass

p1 = Person()

p1.name = "Harry"
p1.occupation = "Developer"
```

Every object requires manual setup.

---

With a constructor:

```python
class Person:

    def __init__(self, name, occupation):
        self.name = name
        self.occupation = occupation
```

Now object creation automatically initializes data.

```python
p1 = Person("Harry", "Developer")
```

---

# 2. Syntax of `__init__`

### Basic Structure

```python
class ClassName:

    def __init__(self):
        # Initialization code
        pass
```

---

### Example

```python
class Person:

    def __init__(self, name, occupation):

        self.name = name
        self.occupation = occupation

        print("A new Person instance has been successfully created!")

    def info(self):
        print(f"{self.name} works as a {self.occupation}")
```

---

## Breakdown

### Constructor Declaration

```python
def __init__(...)
```

Defines the constructor.

---

### Self Reference

```python
self
```

Represents the current object being created.

---

### Parameters

```python
name
occupation
```

Values supplied during object creation.

---

### Instance Attributes

```python
self.name = name
self.occupation = occupation
```

Store data inside the object.

---

# 3. Instantiating Objects with Arguments

Because the constructor expects:

```python
name
occupation
```

those values must be supplied during object creation.

---

### Example

```python

```

---

### Memory Representation

```text

```

---

### Calling Methods


# 4. Understanding the Argument Count

Many beginners notice something strange.

The constructor is defined as:

```python
def __init__(self, name, occupation)
```

which appears to require:

```text
3 Arguments
```

Yet we create objects using:

```python
Person("Harry", "Developer")
```

which only supplies:

```text
2 Arguments
```

---

## Why Does This Work?

Python automatically passes the object itself as the first argument.

---

### What You Write

```python
a = Person("Harry", "Developer")
```

---

### What Python Does Internally

```python
Person.__init__(a, "Harry", "Developer")
```

Notice:

```python
self == a
```

is automatically provided.

---

```text
)
```

# 5. What Happens During Object Creation?

Consider:

```python
a = Person("Harry", "Developer")
```

Python performs the following steps.

---

### Step 1

Allocate memory for object `a`.

```text
Object Created
```

---

### Step 2

Call constructor automatically.

```python
__init__()
```

---

### Step 3

Assign values.

```python
self.name = "Harry"
self.occupation = "Developer"
```

---

### Step 4

Return initialized object.

```text
Object Ready For Use
```

# 6. TypeError Examples

If insufficient arguments are supplied:

```python
a = Person("Harry")
```

Python raises:

```text
TypeError:
missing 1 required positional argument
```

---

If too many arguments are supplied:

```python
a = Person(
    "Harry",
    "Developer",
    "Extra"
)
```

Python raises:

```text
TypeError:
takes 3 positional arguments but 4 were given
```

Remember:

```text
self counts as an argument.
```

# 7. Types of Constructors

Python commonly uses two constructor styles.

---

# A. Parameterized Constructor

A constructor that accepts user-supplied values.

---

### Example

```python
class Person:

    def __init__(self, name, occupation):

        self.name = name
        self.occupation = occupation
```

---

### Usage

```python
a = Person(
    "Harry",
    "Developer"
)

b = Person(
    "Divya",
    "Manager"
)
```

Each object receives unique values.

---

# B. Default Constructor

A constructor that accepts no parameters other than `self`.

Every object starts with identical default values.

---

### Example

```python
class DefaultDemo:

    def __init__(self):

        self.name = "Guest User"
        self.access_level = "Basic"
```

---

### Usage

```python
user = DefaultDemo()

print(user.name)
print(user.access_level)
```

### Output

```text
Guest User
Basic
```

---

# 8. Constructor vs Normal Method

## Constructor

```python
def __init__(self):
```

Runs automatically.

---

## Normal Method

```python
def info(self):
```

Must be called manually.

---

### Example

```python
class Demo:

    def __init__(self):
        print("Constructor Executed")

    def greet(self):
        print("Hello")
```

---

### Execution

```python
obj = Demo()
```

Output:

```text
Constructor Executed
```

---

### Manual Call

```python
obj.greet()
```

Output:

```text
Hello
```

---

# 9. Multiple Instance Attributes

Constructors can initialize many attributes at once.

### Example

```python
class Employee:

    def __init__(
        self,
        name,
        salary,
        department
    ):

        self.name = name
        self.salary = salary
        self.department = department
```

---

### Usage

```python
emp = Employee(
    "Divyanshu",
    100000,
    "AI"
)

print(emp.name)
print(emp.salary)
print(emp.department)
```

---

### Output

```text
Divyanshu
100000
AI
```

---

# 10. Constructor Return Rule

Constructors are not designed to return values.

---

### Incorrect

```python
class Demo:

    def __init__(self):
        return 10
```

Output:

```text
TypeError:
__init__() should return None
```

---

### Correct

```python
class Demo:

    def __init__(self):
        self.value = 10
```

Constructors modify object state.

They do not return data.

---

# Complete Example

```python
class Person:

    def __init__(
        self,
        name,
        occupation
    ):

        self.name = name
        self.occupation = occupation

        print("Person Created")

    def info(self):

        print(
            f"{self.name} works as a {self.occupation}"
        )


a = Person(
    "Harry",
    "Developer"
)

b = Person(
    "Divya",
    "HR Manager"
)

a.info()
b.info()
```

### Output

```text
Person Created
Person Created

Harry works as a Developer
Divya works as a HR Manager
```

# Key Takeaways

### Constructors

✅ Execute automatically

✅ Initialize object data

✅ Improve code organization

---

### `self`

✅ Refers to the current object

✅ Automatically supplied by Python

---

### Parameterized Constructors

✅ Allow unique object states

```python
Person("Harry", "Developer")
```

---

### Default Constructors

✅ Create identical starting configurations

```python
DefaultDemo()
```

---

### Important Rule

You never write:

```python
obj.__init__()
```

directly.

Python automatically calls:

```python
__init__()
```

whenever you create an object using:

```python
ClassName(...)
```

Constructors are the foundation of object initialization and are one of the most important features of Object-Oriented Programming in Python.
