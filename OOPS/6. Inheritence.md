# Inheritance in Python

**Inheritance** is one of the four fundamental pillars of Object-Oriented Programming (OOP).

It allows one class to acquire the attributes and methods of another class, promoting code reuse, maintainability, and extensibility.

Instead of rewriting existing logic, a new class can inherit functionality from an already-tested class and extend it with additional features.

---

# 1. What is Inheritance?

Inheritance is a mechanism that allows a class to derive properties and behaviors from another class.

---

## Terminology

### Parent Class

Also known as:

```text
Base Class
Superclass
```

The class whose features are inherited.

---

### Child Class

Also known as:

```text
Derived Class
Subclass
```

The class that inherits from another class.

---

# Real-World Analogy

Imagine a company.

All employees have:

```text
Name
Employee ID
Department
```

Now suppose the company hires programmers.

A programmer is still an employee.

However, programmers additionally have:

```text
Programming Language
Framework Knowledge
```

Instead of recreating all employee-related code, the Programmer class can inherit from Employee.

---

## Visual Representation

```text
Employee
│
├── name
├── id
└── show_details()

        │
        ▼

Programmer

├── name
├── id
├── show_details()
└── show_language()
```

The child automatically receives everything from the parent.

---

# 2. Why Use Inheritance?

Without inheritance:

```python
class Employee:
    ...
```

```python
class Programmer:
    ...
```

```python
class Manager:
    ...
```

You would repeatedly rewrite:

```text
Name
ID
Address
Department
Salary
```

for every class.

---

This causes:

❌ Code duplication

❌ Maintenance issues

❌ Increased bugs

❌ Violation of DRY principle

---

With inheritance:

```python
class Programmer(Employee):
```

Python automatically reuses existing code.

---

# 3. Basic Syntax

Inheritance is created by placing the parent class inside parentheses.

---

## Syntax

```python
class Parent:
    pass


class Child(Parent):
    pass
```

---

# Example

```python
class Employee:

    def __init__(self, name, id):

        self.name = name
        self.id = id

    def show_details(self):

        print(
            f"The name of Employee {self.id} is {self.name}"
        )


class Programmer(Employee):

    def show_language(self):

        print(
            "The default preferred language is Python."
        )
```

---

# Class Hierarchy

```text
Employee
      ▲
      │
      │ inherits
      │
Programmer
```

# 6. Method Inheritance

The child automatically inherits methods.

---

## Parent

```python
def show_details():
```

---

## Child

```python
e2.show_details()
```

works despite not existing inside Programmer.

---

Why?

Python searches upward through the inheritance chain.

---

# Attribute Lookup Order

When Python encounters:

```python
e2.show_details()
```

it searches:

---

### Step 1

```text
Programmer Class
```

Method found?

```text
No
```

---

### Step 2

```text
Employee Class
```

Method found?

```text
Yes
```

---

### Step 3

Execute method.

---

# Visual Search Path

```text
Programmer

   │
   ▼

show_details ?

   │
   ▼

Employee

   │
   ▼

Method Found
```

# 7. Child Objects Can Access Parent Features

A child object can freely use:

✅ Parent methods

✅ Parent attributes

✅ Parent constructor

---

Example:

```python
e2.show_details()
```

works perfectly.

---

# 8. Parent Objects Cannot Access Child Features

Inheritance is one-directional.

---

### Valid

```python
Programmer
     ↓
Employee
```

Child accesses parent.

---

### Invalid

```python
Employee
     ↑
Programmer
```

Parent cannot access child features.

---

## Example

```python
e1.show_language()
```

Output:

```text
AttributeError
```

---

Why?

Because:

```text
Employee
```

does not contain:

```python
show_language()
```

---

# Visual Explanation

```text
Employee

├── show_details()
```

---

```text
Programmer

├── show_details()
├── show_language()
```

---

Employee only knows:

```python
show_details()
```

It cannot see:

```python
show_language()
```

---

# 9. Constructor Inheritance

Notice:

```python
class Programmer(Employee):
```

contains no constructor.

---

Yet:

```python
e2 = Programmer(
    "Harry",
    430
)
```

works successfully.

---

Why?

Because Python automatically inherits:

```python
Employee.__init__()
```

---

Internally:

```python
Programmer
```

borrows:

```python
Employee.__init__()
```

until its own constructor is defined.

---

# 10. Complete Example

```python
class Employee:

    def __init__(self, name, id):

        self.name = name
        self.id = id

    def show_details(self):

        print(
            f"The name of Employee {self.id} is {self.name}"
        )


class Programmer(Employee):

    def show_language(self):

        print(
            "The default preferred language is Python."
        )


e1 = Employee(
    "Rohan Das",
    400
)

e1.show_details()


e2 = Programmer(
    "Harry",
    430
)

e2.show_details()
e2.show_language()
```

---

### Output

```text
The name of Employee 400 is Rohan Das

The name of Employee 430 is Harry

The default preferred language is Python.
```

# Types of Inheritance

Python supports several inheritance structures.

---

# 1. Single Inheritance

One child inherits from one parent.

```text
Employee
    │
    ▼
Programmer
```

---

## Example

```python
class Programmer(Employee):
    pass
```

---

# 2. Multiple Inheritance

One child inherits from multiple parents.

```text
ParentA
    │
    ├─────► Child
    │
ParentB
```

---

## Example

```python
class Child(
    ParentA,
    ParentB
):
    pass
```

---

# 3. Multilevel Inheritance

Inheritance chain.

```text
Grandparent
      │
      ▼
Parent
      │
      ▼
Child
```

---

## Example

```python
class A:
    pass

class B(A):
    pass

class C(B):
    pass
```

---

# 4. Hierarchical Inheritance

Multiple children share one parent.

```text
        Employee
        /     \
       /       \
Programmer   Manager
```

---

## Example

```python
class Programmer(Employee):
    pass

class Manager(Employee):
    pass
```
