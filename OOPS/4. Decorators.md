# Decorators in Python

A **Decorator** is an advanced and highly flexible design pattern in Python that allows you to modify, extend, or enhance the behavior of a function or method **without changing its original source code**.

Formally, a decorator is a **higher-order function** that:

1. Accepts another function as an argument.
2. Adds additional functionality.
3. Returns a new modified function.

Decorators are one of Python's most powerful features and are heavily used in frameworks such as Flask, Django, FastAPI, and many internal Python libraries.

---

# 1. The Core Problem

Imagine a large application containing hundreds of functions.

```python
def login():
    pass

def logout():
    pass

def process_payment():
    pass

def generate_report():
    pass
```

Now suppose your team requests:

```text
Print "Good Morning" before every function runs.

Print "Thanks for using this function"
after every function finishes.
```

---

## The Inefficient Approach

You could manually modify every function.

```python
def hello():

    print("Good Morning")

    print("Hello World")

    print("Thanks for using this function")
```

However:

❌ Code duplication

❌ Difficult maintenance

❌ Violates DRY (Don't Repeat Yourself)

---

# 2. The Decorator Solution

Instead of modifying every function individually, we can create one reusable wrapper.

---

## Basic Decorator

```python
def greet(fx):

    def mfx():

        print("Good Morning")

        fx()

        print("Thanks for using this function")

    return mfx
```

---

## Understanding the Flow

### Step 1

Decorator receives a function.

```python
greet(hello)
```

```text
fx = hello
```

---

### Step 2

Creates a wrapper.

```python
def mfx():
```

---

### Step 3

Executes additional code.

```python
print("Good Morning")
```

---

### Step 4

Runs original function.

```python
fx()
```

---

### Step 5

Executes additional code.

```python
print("Thanks for using this function")
```

---

### Step 6

Returns wrapper.

```python
return mfx
```

---

# 3. Manual Decoration

Before learning the `@` symbol, let's see what decorators actually do.

---

### Original Function

```python
def hello():
    print("Hello World")
```

---

### Decorating Manually

```python
hello = greet(hello)
```

Now:

```python
hello()
```

actually executes:

```python
mfx()
```

instead.

---

### Output

```text
Good Morning
Hello World
Thanks for using this function
```

---

# 4. Syntactic Sugar: The `@` Operator

Python provides a cleaner syntax.

Instead of:

```python
hello = greet(hello)
```

we write:

```python
@greet
def hello():
    print("Hello World")
```

---

## Complete Example

```python
def greet(fx):

    def mfx():

        print("Good Morning")

        fx()

        print("Thanks for using this function")

    return mfx


@greet
def hello():
    print("Hello World")


hello()
```

---

### Output

```text
Good Morning
Hello World
Thanks for using this function
```

---

# What Python Does Internally

This:

```python
@greet
def hello():
    print("Hello World")
```

is automatically converted into:

```python
def hello():
    print("Hello World")

hello = greet(hello)
```

---

```text
   ▼

   │
   ├── Good Morning

   ├── Original Function
   │
   └── Thanks For Using This Function

   ▼

Output
```

# 5. The Problem with Arguments

Consider:

```python
def add(a, b):
    print(a + b)
```

If we apply the previous decorator:

```python
@greet
def add(a, b):
    print(a + b)
```

and call:

```python
add(5, 7)
```

Python raises an error.

---

### Why?

Our wrapper was defined as:

```python
def mfx():
```

which accepts:

```text
0 arguments
```

but:

```python
add(5, 7)
```

supplies:

```text
2 arguments
```

---

# 6. Universal Decorators with `*args` and `**kwargs`

To support any function signature, use:

```python
*args
**kwargs
```

---

## Improved Decorator

```python
def greet(fx):

    def mfx(*args, **kwargs):

        print("Good Morning")

        result = fx(*args, **kwargs)

        print("Thanks for using this function")

        return result

    return mfx
```

---

# Example

```python
@greet
def add(a, b):

    print(
        f"The sum calculation is: {a + b}"
    )


add(5, 7)
```

---

### Output

```text
Good Morning
The sum calculation is: 12
Thanks for using this function
```

---

# 7. Returning Values from Decorated Functions

Decorators should preserve return values.

---

### Example

```python
def greet(fx):

    def mfx(*args, **kwargs):

        print("Starting Function")

        result = fx(*args, **kwargs)

        print("Function Finished")

        return result

    return mfx
```

---

### Target Function

```python
@greet
def multiply(a, b):

    return a * b
```

---

### Usage

```python
answer = multiply(5, 4)

print(answer)
```

---

### Output

```text
Starting Function
Function Finished
20
```

# 8. Real-World Example: Execution Timer

One common use case is measuring execution time.

---

```python
import time

def timer(func):

    def wrapper(*args, **kwargs):

        start = time.time()

        result = func(*args, **kwargs)

        end = time.time()

        print(
            f"Execution Time: {end - start}"
        )

        return result

    return wrapper
```

---

### Usage

```python
@timer
def calculate():

    total = 0

    for i in range(1000000):
        total += i

    return total


calculate()
```

---

### Output

```text
Execution Time: 0.02
```

(actual value varies)

# 9. Real-World Example: Authentication

Web frameworks use decorators extensively.

---

```python
def login_required(func):
```

---

### Output

# 10. Multiple Decorators

A function can have multiple decorators.

---

### Example

```python
def decorator1(func):

    def wrapper():
        print("Decorator 1 Start")

        func()

        print("Decorator 1 End")

    return wrapper


def decorator2(func):

    def wrapper():
        print("Decorator 2 Start")

        func()

        print("Decorator 2 End")

    return wrapper
```

---

### Applying Multiple Decorators

```python
@decorator1
@decorator2
def hello():

    print("Hello")
```

---

### Execution Order

Python interprets:

```python
hello = decorator1(
            decorator2(
                hello
            )
        )
```

---

### Output

```text
Decorator 1 Start
Decorator 2 Start
Hello
Decorator 2 End
Decorator 1 End
```

---

# Decorator Anatomy

```python
def decorator(func):

    def wrapper(*args, **kwargs):

        # Extra logic before

        result = func(
            *args,
            **kwargs
        )

        # Extra logic after

        return result

    return wrapper
```

---

# Summary Table

| Component | Purpose |
|------------|----------|
| Decorator | Modifies another function |
| Higher-Order Function | Accepts a function as input |
| Wrapper Function | Contains added behavior |
| `@decorator` | Applies decorator automatically |
| `*args` | Handles positional arguments |
| `**kwargs` | Handles keyword arguments |
| `return result` | Preserves original return value |
