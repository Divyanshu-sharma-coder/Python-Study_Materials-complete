# Operator Overloading in Python

## 1. What is Operator Overloading?

Operator overloading is a feature in Python that allows you to change the default behavior of mathematical, logical, or comparison operators (like `+`, `-`, `*`, `/`) when they are applied to user-defined objects.

For example:

```python
5 + 5   # 10
"a" + "b"  # "ab"
```

But for custom objects like vectors:

```python
v1 + v2   # ❌ Not defined by default
```

So we define how operators should behave for our class.

---

## 2. Magic Methods (Dunder Methods)

Python uses special methods called **Magic Methods** (or Dunder Methods) to overload operators.

| Operator | Method | Meaning |
|:--------:|:------:|---------|
| `+` | `__add__` | Addition |
| `-` | `__sub__` | Subtraction |
| `*` | `__mul__` | Multiplication |
| `/` | `__truediv__` | Division |
| `//` | `__floordiv__` | Floor Division |

---

## 3. Practical Example: Vector Addition

```python
class Vector:
    def __init__(self, i, j, k):
        self.i = i
        self.j = j
        self.k = k

    def __str__(self):
        return f"{self.i}i + {self.j}j + {self.k}k"

    def __add__(self, x):
        return Vector(self.i + x.i,
                      self.j + x.j,
                      self.k + x.k)

# Creating vectors
v1 = Vector(3, 5, 6)
v2 = Vector(1, 2, 9)

print(f"Vector 1: {v1}")
print(f"Vector 2: {v2}")

# Operator overloading in action
print(f"Sum:      {v1 + v2}")
```

---

## 4. Code Breakdown

### ❌ Without Operator Overloading
If `__add__` is not defined:

```
TypeError: unsupported operand type(s) for +: 'Vector' and 'Vector'
```

Because Python doesn’t know how to add custom objects.

---

### ✔ With Operator Overloading

When you do:

```python
v1 + v2
```

Python internally calls:

```python
v1.__add__(v2)
```

So:

```
(3i + 5j + 6k) + (1i + 2j + 9k)
= 4i + 7j + 15k
```

---

## 5. Terminal Output

```
Vector 1: 3i + 5j + 6k
Vector 2: 1i + 2j + 9k
Sum:      4i + 7j + 15k
```

## 6. Key Takeaways

### ✔ Operator Overloading
- Lets you define behavior of operators for custom objects
- Makes code intuitive and readable

### ✔ `__str__`
- Controls how object is displayed in print()

### ✔ `__add__`
- Defines how `+` works for your class
