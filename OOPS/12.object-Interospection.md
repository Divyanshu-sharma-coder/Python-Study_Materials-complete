# Object Introspection in Python: `dir()`, `__dict__`, and `help()`

When working with large codebases, third-party libraries, frameworks, or unfamiliar classes, one of the biggest challenges is figuring out:

- What methods are available?
- What attributes exist?
- What data is stored inside an object?
- How is a class supposed to be used?

Instead of searching documentation every time, Python provides powerful **Object Introspection** tools.

Object Introspection is Python's ability to examine objects during runtime and reveal their structure, attributes, methods, and documentation.

---

# What is Object Introspection?

Object Introspection means:

> "Examining an object while the program is running."

Python lets us inspect:

- Variables
- Functions
- Classes
- Objects
- Modules

without needing access to their source code.

---

# Why is Introspection Important?

Imagine someone gives you:

```python
obj = SomeComplexLibraryObject()
```

You have never seen this class before.

Questions immediately arise:

```text
What methods does it have?

What attributes exist?

How do I use it?

What data is stored inside it?
```

Python provides built-in tools to answer these questions instantly.

---

# The Three Most Common Introspection Tools

| Tool | Purpose |
|--------|----------|
| `dir()` | Show available attributes and methods |
| `__dict__` | Show stored instance variables |
| `help()` | Show documentation |

---

# 1. The `dir()` Function

The `dir()` function returns a list containing every accessible attribute and method available to an object.

Think of it as:

```text
"What can this object do?"
```

---

# Basic Syntax

```python
dir(object)
```

---

# Example with a List

```python
numbers = [1, 2, 3]

print(dir(numbers))
```

---

```python
    'insert',
    'pop',

    'reverse',

]
```

# 2. The `__dict__` Attribute

While:

```python
dir()
```

shows everything an object can do,

sometimes you only want to see:

```text
What data is actually stored inside this object?
```

For that purpose Python provides:

```python
__dict__
```

---

# What is `__dict__`?

`__dict__` is a built-in dictionary that stores all instance variables belonging to an object.

Think of it as:

```text
Object Memory Storage
```

---

# Example

```python
class Person:

    def __init__(self, name, age):

        self.name = name

        self.age = age

        self.version = 1
```

---

# Creating Object

```python
p = Person(
    "John",
    30
)
```

---

# Viewing Internal Storage

```python
print(p.__dict__)
```

Output:

```python
{
    'name': 'John',
    'age': 30,
    'version': 1
}
```

---

# What's Happening?

Python internally stores:

```python
self.name
```

---

```python
self.age
```

---

```python
self.version
```

inside a dictionary.

---

# Internal View

```python
p.__dict__
```

Equivalent to:

```python
{
    "name": "John",
    "age": 30,
    "version": 1
}
```

---

# Visual Representation

```text
Person Object

┌─────────────┐
│ name=John   │
│ age=30      │
│ version=1   │
└─────────────┘
```

---

```python
__dict__
```

simply exposes that storage.

---

# Modifying Through __dict__

Because it's a dictionary:

```python
p.__dict__["age"] = 35
```

---

```python
print(p.age)
```

Output:

```text
35
```

---

Although possible, modifying objects through `__dict__` is rarely recommended in production code.

---

# Difference Between dir() and __dict__

---

## dir()

Shows:

```text
Methods
Properties
Inherited Attributes
Special Methods
```

---

## __dict__

Shows only:

```text
Actual Stored Variables
```

---

# Example

```python
print(dir(p))
```

Output:

```python
[
    '__class__',
    '__dict__',
    '__str__',
    'age',
    'name',
    'version'
]
```

---

```python
print(p.__dict__)
```

Output:

```python
{
    'name': 'John',
    'age': 30,
    'version': 1
}
```

---

# Visual Comparison

```text

Entire Blueprint
```

---

```text
__dict__()

Stored Data Only
```

# 3. The `help()` Function

Suppose:

```python
dir()
```

shows:

```python
append
remove
insert
sort
```

But you don't know:

```text
What arguments do they need?

What do they return?

How are they used?
```

This is where:

```python
help()
```

becomes useful.

---

# What Does help() Do?

`help()` generates runtime documentation.

It displays:

- Method descriptions
- Class descriptions
- Function signatures
- Parameters
- Docstrings

---

# Basic Syntax

```python
help(object)
```

---

# Example

```python
help(list)
```

---

Partial Output:

```text
Help on class list:

append(...)
    Append object to the end of the list.

clear(...)
    Remove all items from list.

copy(...)
    Return a shallow copy of the list.
```

---

Now Python acts like a built-in manual.

---

# Example with User-Defined Class

```python
class Person:

    """
    Represents a human being.
    """

    def __init__(self, name, age):
        self.name = name
        self.age = age
```

---

```python
help(Person)
```

Output includes:

```text
Help on class Person

Represents a human being.
```

plus constructor information.

---

# What is a Docstring?

A Docstring is a built-in documentation string.

---

Example:

```python
class Calculator:

    """
    Simple calculator utility.
    """
```

---

or

```python
def add(a, b):
    """
    Returns sum of two numbers.
    """
```

---

`help()` automatically reads and displays these.

---

# Visual Representation

```text
Object

   │
   ▼

help()

   │
   ▼

Documentation
Manual
Usage Guide
```

# Full Example

```python
class Person:

    """
    Represents a person.
    """

    def __init__(self, name, age):

        self.name = name

        self.age = age

    def greet(self):

        print(
            f"Hello {self.name}"
        )
```

---

```python
p = Person(
    "John",
    30
)
```

---

# dir()

```python
print(dir(p))
```

Shows:

```text
Available methods and attributes
```

---

# __dict__

```python
print(p.__dict__)
```

Shows:

```text
Stored data values
```

---

# help()

```python
help(Person)
```

Shows:

```text
Documentation and usage information
```

# Introspection Comparison Matrix

| Tool | Returns | Purpose |
|--------|---------|----------|
| `dir(obj)` | List | Discover available methods and attributes |
| `obj.__dict__` | Dictionary | View stored instance variables |
| `help(obj)` | Documentation | Learn how to use an object |

---

# Real-World Workflow

When exploring an unknown object:

---

### Step 1

Use:

```python
dir(obj)
```

to see what exists.

---

### Step 2

Use:

```python
obj.__dict__
```

to inspect stored data.

---

### Step 3

Use:

```python
help(obj)
```

to learn how everything works.

---

# Example

```python
import math
```

---

### Discover Functions

```python
dir(math)
```

---

### Learn Function Usage

```python
help(math.sqrt)
```

---

Output:

```text
sqrt(x)

Return the square root of x.
```
