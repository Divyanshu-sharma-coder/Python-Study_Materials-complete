# Getters and Setters in Python (`@property`)

In many Object-Oriented Programming languages, **Getters** and **Setters** are special methods used to access and modify an object's internal data while maintaining encapsulation and control.

Python provides a cleaner and more elegant approach through the **`@property`** decorator, allowing methods to behave like normal attributes.

This enables you to expose calculated values, perform validation, and maintain backward compatibility while keeping the syntax simple and intuitive.

---

# 1. The Core Motivation

Suppose you have an internal variable:

```python
self._value
```

and you want users to access a value that is always:

```text
10 × _value
```

A beginner approach might be:

```python
obj.get_ten_x_value()
```

However, this feels like a method call rather than a property.

Python allows you to expose it like:

```python
obj.ten_x_value
```

while still executing code behind the scenes.

---

# 2. Understanding the `@property` Decorator

The `@property` decorator converts a method into a read-only attribute.

---

## Basic Syntax

```python
class MyClass:

    @property
    def attribute_name(self):
        return value
```

---

## Example: Creating a Getter

```python
class MyClass:

    def __init__(self, value):
        self._value = value

    @property
    def ten_x_value(self):
        return 10 * self._value
```

---

# Using the Class

```python
obj = MyClass(10)

print(obj.ten_x_value)
```

### Output

```text
100
```

Notice:

```python
obj.ten_x_value
```

looks like an attribute access.

There are:

```text
No Parentheses
```

yet Python still executes the method.

---

# What Happens Internally?

When Python sees:

```python
obj.ten_x_value
```

it secretly converts it into:

```python
obj.ten_x_value()
```

behind the scenes.

The user experiences:

```text
Attribute Syntax
```

while Python executes:

```text
Method Logic
```

---

# Visual Representation

```text
User Writes

obj.ten_x_value

        │
        ▼

@property Method Executes

        │
        ▼

return 10 * self._value

        │
        ▼

100
```

---

# 3. Why Use `_value` Instead of `value`?

Notice:

```python
self._value
```

contains a leading underscore.

---

## Meaning

```python
_value
```

is a Python convention indicating:

```text
"Internal Use Only"
```

It is not truly private.

It simply tells developers:

```text
Do not access this directly unless necessary.
```

---

# 4. Read-Only Properties

A property created using only:

```python
@property
```

is read-only.

---

### Example

```python
class MyClass:

    def __init__(self, value):
        self._value = value

    @property
    def ten_x_value(self):
        return self._value * 10
```

---

### Usage

```python
obj = MyClass(10)

print(obj.ten_x_value)
```

Output:

```text
100
```

---

### Attempting Modification

```python
obj.ten_x_value = 67
```

Output:

```text
AttributeError:
can't set attribute
```

---

# Why Does This Error Occur?

Because Python only knows:

```python
How To Read
```

the property.

It does not know:

```python
How To Write
```

to it.

---

# 5. Creating a Setter

To allow modification, define a matching setter.

---

## Syntax

```python
@property
def attribute(self):
    ...

@attribute.setter
def attribute(self, value):
    ...
```

The names must be identical.

---

# Example

```python
class MyClass:

    def __init__(self, value):
        self._value = value

    @property
    def ten_x_value(self):
        return 10 * self._value

    @ten_x_value.setter
    def ten_x_value(self, new_value):
        self._value = new_value / 10
```

---

# Using the Setter

```python
obj = MyClass(10)

obj.ten_x_value = 67
```

Python automatically executes:

```python
ten_x_value.setter(...)
```

behind the scenes.

---

# Current Object State

```python
print(obj._value)
```

Output:

```text
6.7
```

---

# Verification

```python
print(obj.ten_x_value)
```

Output:

```text
67.0
```

---

# Internal Execution Flow

When you write:

```python
obj.ten_x_value = 67
```

Python secretly executes:

```python
obj.ten_x_value(67)
```

through the setter.

---

## Visual Flow

```text
obj.ten_x_value = 67

          │
          ▼

Setter Executes

          │
          ▼

self._value = 67 / 10

          │
          ▼

self._value = 6.7
```

---

# 6. Complete Example

```python
class MyClass:

    def __init__(self, value):
        self._value = value

    @property
    def ten_x_value(self):
        return self._value * 10

    @ten_x_value.setter
    def ten_x_value(self, new_value):
        self._value = new_value / 10


obj = MyClass(10)

print(obj.ten_x_value)

obj.ten_x_value = 67

print(obj._value)

print(obj.ten_x_value)
```

---

### Output

```text
100
6.7
67.0
```

---

# 7. Real Use Case: Data Validation

One major advantage of setters is validation.

---

## Example

```python
class Employee:

    def __init__(self):
        self._salary = 0

    @property
    def salary(self):
        return self._salary

    @salary.setter
    def salary(self, value):

        if value < 0:
            print("Salary cannot be negative.")
            return

        self._salary = value
```

---

### Usage

```python
emp = Employee()

emp.salary = 50000

print(emp.salary)
```

Output:

```text
50000
```

---

### Invalid Input

```python
emp.salary = -1000
```

Output:

```text
Salary cannot be negative.
```

---

# 8. Real Use Case: Computed Properties

Properties are often used for values derived from other attributes.

---

## Example

```python
class Circle:

    def __init__(self, radius):
        self.radius = radius

    @property
    def area(self):
        return 3.14159 * self.radius ** 2
```

---

### Usage

```python
c = Circle(5)

print(c.area)
```

Output:

```text
78.53975
```

---

Notice:

```python
c.area
```

looks like an attribute.

Yet a calculation occurs every time.

---

# 9. Backward Compatibility

Properties allow developers to change internal implementations without breaking existing code.

---

## Initial Version

```python
class User:

    def __init__(self):
        self.name = "Harry"
```

Users access:

```python
user.name
```

---

## Future Version

Suppose you later need extra logic.

Instead of changing:

```python
user.name
```

everywhere,

you can write:

```python
@property
def name(self):
    ...
```

and existing code continues working.

---

# Why This Matters

Large applications may contain:

```text
Thousands of references
```

to a property.

Properties allow upgrades without rewriting everything.

---

# 10. Getter vs Setter vs Normal Attribute

| Feature | Normal Attribute | Getter (`@property`) | Setter (`@property.setter`) |
|----------|----------|----------|----------|
| Read Value | ✅ | ✅ | N/A |
| Write Value | ✅ | ❌ | ✅ |
| Runs Logic | ❌ | ✅ | ✅ |
| Supports Validation | ❌ | ❌ | ✅ |
| Computed Values | ❌ | ✅ | ❌ |

---

# Full Professional Example

```python
class Employee:

    def __init__(self, salary):
        self._salary = salary

    @property
    def salary(self):
        return self._salary

    @salary.setter
    def salary(self, value):

        if value < 0:
            raise ValueError(
                "Salary cannot be negative"
            )

        self._salary = value


emp = Employee(50000)

print(emp.salary)

emp.salary = 75000

print(emp.salary)
```

---

### Output

```text
50000
75000
```

---

# Summary Table

| Concept | Description |
|----------|------------|
| `@property` | Converts a method into a read-only attribute |
| Getter | Retrieves or computes a value |
| Setter | Controls how a value is modified |
| `_attribute` | Internal variable naming convention |
| Validation | Prevents invalid data assignments |
| Computed Property | Calculates values dynamically |
| Backward Compatibility | Allows internal changes without changing external code |

### Getter

```python

```

Allows:

```python
obj.attribute
```

instead of:


---

### Setter


Allows:

```python
obj.attribute = value
```

while running custom logic.

✅ Computed Properties

✅ Backward Compatibility
