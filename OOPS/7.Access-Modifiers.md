# Access Modifiers in Python: Public, Protected, and Private

Access Modifiers are mechanisms used in Object-Oriented Programming (OOP) to control the visibility and accessibility of attributes and methods.

In languages such as Java, C++, and C#, access modifiers are enforced strictly by the compiler.

Python takes a different approach.

Instead of enforcing strict access restrictions, Python relies heavily on **naming conventions** and **developer discipline**.

This philosophy follows the famous Python principle:

```text
"We are all consenting adults here."
```

Python assumes developers will respect naming conventions rather than requiring rigid restrictions.

---

# 1. What Are Access Modifiers?

Access Modifiers determine:

```text
Who can access an attribute?
Who can modify it?
Where should it be used?
```

---

## Main Types

Python recognizes three commonly used access levels:

| Modifier | Syntax |
|-----------|----------|
| Public | `variable` |
| Protected | `_variable` |
| Private | `__variable` |

---

# Important Python Reality

Unlike Java or C++:

```text
Python does NOT truly enforce access modifiers.
```

Instead:

```text
Public     → No restrictions

Protected  → Convention only

Private    → Name Mangling
```

---

# 2. Public Access Modifier

Public is Python's default access level.

Every attribute and method is public unless specified otherwise.

---

## Example

```python
class Employee:

    def __init__(self, name):
        self.name = name
```

---

## Usage

```python
e = Employee("Harry")

print(e.name)
```

### Output

```text
Harry
```

---

## Modification

Public attributes can be modified directly.

```python
e.name = "Rohan"

print(e.name)
```

### Output

```text
Rohan
```

---

# Visual Representation

```text
Employee Object

name = Harry

      ▲
      │
      │ Accessible
      │
Outside Code
```

---

# Characteristics of Public Members

✅ Readable everywhere

✅ Writable everywhere

✅ No restrictions

---

# 3. Protected Access Modifier

Protected members use a single underscore.

---

## Syntax

```python
_variable
```

---

## Example

```python
class Student:

    def __init__(self, name):

        self._name = name
```

---

# What Does `_` Mean?

The single underscore means:

```text
"This attribute is intended for internal use."
```

It acts as a warning to other developers.

---

## Important

Python performs:

```text
NO restriction
```

on protected members.

---

# Accessing Protected Variables

```python
obj = Student("Harry")

print(obj._name)
```

### Output

```text
Harry
```

Even though it works, it is considered poor design.

---

# Why Use Protected Members?

Protected members are primarily intended for:

```text
Class
+
Subclasses
```

---

# Example

```python
class Student:

    def __init__(self, name):

        self._name = name


class Subject(Student):

    def show_name(self):

        print(self._name)
```

---

## Usage

```python
obj = Subject("Rohan")

obj.show_name()
```

### Output

```text
Rohan
```

---

# Visual Representation

```text
Student

└── _name

        ▲
        │
        │ Intended Access
        │
        ▼

Subject
```

---

# Protected Access Summary

### Intended Usage

```text
Class
+
Child Classes
```

---

### Actual Python Behavior

```text
Anyone can still access it.
```

---

# 4. Private Access Modifier

Private members use a double underscore.

---

## Syntax

```python
__variable
```

---

## Example

```python
class Employee:

    def __init__(self, name):

        self.__name = name
```

---

# Attempting Direct Access

```python
e = Employee("Harry")

print(e.__name)
```

### Output

```text
AttributeError
```

---

Many beginners think:

```text
Private means inaccessible.
```

Not exactly.

Python uses a mechanism called:

```text
Name Mangling
```

---

# 5. Understanding Name Mangling

When Python encounters:

```python
self.__name
```

it secretly renames it.

---

## Original

```python
__name
```

---

## Internally Converted To

```python
_Employee__name
```

Pattern:

```text
_ClassName__VariableName
```

---

# Visual Representation

```text
Source Code

__name

       │
       ▼

Python Compiler

       │
       ▼

_Employee__name
```

---

# Why Does Python Do This?

The goal is:

```text
Prevent accidental overwriting
```

during inheritance.

---

Not:

```text
Provide military-grade security.
```

---

# Accessing Mangled Variables

Although direct access fails:

```python
print(e.__name)
```

you can still access:

```python
print(e._Employee__name)
```

---

### Output

```text
Harry
```

---

# Complete Example

```python
class Employee:

    def __init__(self, name):

        self.__name = name


e = Employee("Harry")

print(e._Employee__name)
```

### Output

```text
Harry
```

---

# 6. Why Private Variables Exist

Private members help:

✅ Avoid accidental modification

✅ Prevent subclass conflicts

✅ Signal implementation details

✅ Improve encapsulation

---

# Example Problem

Imagine:

```python
class Employee:

    def __init__(self):

        self.__id = 101
```

and later:

```python
class Programmer(Employee):

    def __init__(self):

        self.__id = 999
```

Without name mangling:

```text
Conflict Possible
```

---

With name mangling:

```python
_Employee__id
```

and

```python
_Programmer__id
```

become different variables.

---

# Visual Representation

```text
Employee

└── _Employee__id
```

---

```text
Programmer

└── _Programmer__id
```

No collision occurs.

# 7. Comparing Public, Protected, and Private

---

## Public Example

```python
class Demo:

    def __init__(self):

        self.name = "Harry"
```

Access:

```python
obj.name
```

✅ Allowed

---

## Protected Example

```python
class Demo:

    def __init__(self):

        self._name = "Harry"
```

Access:

```python
obj._name
```

✅ Allowed

⚠ Discouraged

---

## Private Example

```python
class Demo:

    def __init__(self):

        self.__name = "Harry"
```

Access:

```python
obj.__name
```

❌ AttributeError

---

Indirect Access:

```python
obj._Demo__name
```

✅ Allowed

---

# 8. Methods Can Also Be Public, Protected, and Private

---

## Public Method

```python
class Demo:

    def show(self):

        print("Public")
```

Usage:

```python
obj.show()
```

---

## Protected Method

```python
class Demo:

    def _show(self):

        print("Protected")
```

Usage:

```python
obj._show()
```

Works but discouraged.

---

## Private Method

```python
class Demo:

    def __show(self):

        print("Private")
```

Usage:

```python
obj.__show()
```

Output:

```text
AttributeError
```

---

# 9. Complete Demonstration

```python
class Employee:

    def __init__(self):

        self.public = "Public"

        self._protected = "Protected"

        self.__private = "Private"


obj = Employee()

print(obj.public)

print(obj._protected)

print(obj._Employee__private)
```

---

### Output

```text
Public
Protected
Private
```

# Summary Matrix

| Modifier | Syntax | Accessible Outside Class | Python Enforcement |
|-----------|----------|-------------------------|-------------------|
| Public | `variable` | ✅ Yes | None |
| Protected | `_variable` | ✅ Yes | Convention Only |
| Private | `__variable` | ❌ Direct Access Blocked | Name Mangling |

Python does not truly enforce access restrictions.

