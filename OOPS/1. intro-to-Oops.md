# Introduction to Object-Oriented Programming (OOPs) in Python

**Object-Oriented Programming (OOPs)** is a programming paradigm that organizes software around **Objects** rather than individual functions and variables.

Instead of writing programs as a collection of separate instructions, OOP groups related data and behavior together into logical units.

Modern software systems, games, web applications, operating systems, and AI frameworks heavily rely on Object-Oriented Programming.

---

# 1. Why Use OOPs? (Procedural vs OOPs)

Before understanding OOPs, it is important to understand the problem it solves.

---

## A. The Procedural Limitation

In **Procedural Programming**, programs are built using:

- Variables
- Functions
- Loops
- Conditional Statements

Suppose we want to track information about a shop owner.

### Example

```python
sales_1 = 6000
profit_1 = 2000
ad_spend_1 = 1000
```

Now suppose another shop owner joins.

```python
sales_2 = 8500
profit_2 = 4000
ad_spend_2 = 1500
```

And another:

```python
sales_3 = 3100
profit_3 = 900
ad_spend_3 = 400
```

---

### The Problem

As the system grows:

```text
sales_1
sales_2
sales_3
sales_4
...
sales_1000
```

You end up managing thousands of disconnected variables.

This causes:

- Poor organization
- Code duplication
- Difficult maintenance
- Increased bugs

---

## B. The OOPs Resolution

OOP allows us to model real-world entities directly.

Instead of creating separate variables, we create an object.

### Conceptual Example

```python
rajiv.sales = 6000
rajiv.profit = 2000

vikrant.sales = 8500
vikrant.profit = 4000
```

Now all data belonging to Rajiv stays inside the Rajiv object.

Similarly, all data belonging to Vikrant stays inside the Vikrant object.

---

### Real-World Example: Video Games

Without OOP:

```python
enemy_1_health
enemy_1_weapon

enemy_2_health
enemy_2_weapon

enemy_3_health
enemy_3_weapon
```

With OOP:

```python
enemy.health
enemy.weapon
enemy.attack()
```

Every enemy follows the same structure while maintaining unique values.

---

# 2. The Core Concept: Classes vs Objects

Object-Oriented Programming revolves around two concepts:

1. Classes
2. Objects

---

## A. Class (Blueprint / Template)

A **Class** is a blueprint describing:

- What data an object should contain
- What actions an object can perform

A class does not represent a real object.

It is simply a design template.

---

### Example

Imagine the blueprint for a car.

The blueprint defines:

```text
Car
 ├── Color
 ├── Brand
 ├── Speed
 └── Start Engine()
```

The blueprint itself is not a real car.

It only describes what a car should have.

---

## B. Object (Instance)

An **Object** is a real instance created from a class.

Objects occupy memory and contain actual values.

---

### Example

```text
Class:
Car

Objects:

Car 1
Color = Red
Brand = BMW

Car 2
Color = Black
Brand = Mercedes

Car 3
Color = White
Brand = Audi
```

All objects follow the same blueprint but store different values.

---

# Railway Reservation Form Analogy

Imagine an empty railway reservation form.

The form contains blank fields:

```text
Passenger Name
Age
Train Number
Destination
```

This blank form represents a **Class**.

---

Now suppose three passengers fill out the form:

### Passenger 1

```text
Name: Harry
Age: 25
Destination: Delhi
```

### Passenger 2

```text
Name: Tom
Age: 30
Destination: Mumbai
```

### Passenger 3

```text
Name: Shubham
Age: 21
Destination: Jaipur
```

These completed forms are **Objects**.

---

### Summary

```text
Blank Form
     ↓
   Class

Filled Forms
     ↓
  Objects
```

# 3. Creating Your First Class

### Class Definition

```python
class Employee:
    pass
```

The keyword:

```python
class
```

creates a class.

`pass` is used as a placeholder.

---

### Creating Objects

```python
class Employee:
    pass

harry = Employee()
tom = Employee()
```

Now:

```text
harry → Object
tom   → Object
```

created from the same class.

---

# 4. Adding Data to Objects

Objects can store data using attributes.

```python
class Employee:
    pass

harry = Employee()

harry.name = "Harry"
harry.salary = 50000
harry.department = "AI"
```

---

### Accessing Data

```python
print(harry.name)
print(harry.salary)
```

Output:

```text
Harry
50000
```

---

# 5. Adding Behavior Using Methods

Objects do not only store data.

They can also perform actions.

These actions are called **Methods**.

---

### Example

```python
class Employee:

    def greet(self):
        print("Welcome Employee")

harry = Employee()

harry.greet()
```

Output:

```text
Welcome Employee
```

# 6. The Four Core Pillars of OOPs

Object-Oriented Programming is built upon four foundational principles.

---

## 1. Encapsulation

### Definition

Encapsulation means:

> Bundling data and methods together inside a single unit.

---

### Example

```python
class BankAccount:

    def __init__(self):
        self.balance = 0

    def deposit(self, amount):
        self.balance += amount
```

The balance and operations belong to the same object.

---

### Benefits

- Better security
- Better organization
- Reduced accidental modifications

---

## 2. Inheritance

### Definition

Inheritance allows one class to acquire properties and methods from another class.

---

### Example

```python
class Vehicle:

    def start(self):
        print("Vehicle Started")


class Car(Vehicle):
    pass
```

Now:

```python
car = Car()

car.start()
```

Output:

```text
Vehicle Started
```

The Car class inherited functionality from Vehicle.

---

### Bicycle Analogy

```text
Bicycle
   ↓
Electric Bicycle
```

The electric bicycle inherits:

- Wheels
- Seat
- Handlebars

and simply adds:

- Battery
- Motor

without rebuilding everything.

---

## 3. Polymorphism

### Definition

Polymorphism means:

> One interface, many implementations.

The same method name can behave differently depending on the object.

---

### Example

```python
class Dog:

    def sound(self):
        print("Bark")


class Cat:

    def sound(self):
        print("Meow")
```

Usage:

```python
dog = Dog()
cat = Cat()

dog.sound()
cat.sound()
```

Output:

```text
Bark
Meow
```

Same method:

```python
sound()
```

Different behaviors.

---

## 4. Abstraction

### Definition

Abstraction hides unnecessary complexity and exposes only essential functionality.

---

### Example

When driving a car:

You interact with:

```text
Steering Wheel
Brake
Accelerator
```

You do not need to know:

```text
Fuel Injection Logic
Combustion Cycle
Transmission Internals
```

Those details are hidden.

---

### Software Example

```python
file = open("data.txt")
```

You use:

```python
open()
```

without knowing how Python communicates with the operating system internally.

That hidden complexity is abstraction.

---

# Procedural vs OOP Comparison

| Feature | Procedural Programming | OOP Programming |
|----------|----------------------|-----------------|
| Focus | Functions | Objects |
| Data Organization | Separate Variables | Bundled Inside Objects |
| Reusability | Limited | High |
| Scalability | Difficult | Easy |
| Real-World Modeling | Weak | Strong |
| Code Maintenance | Hard | Easier |

### Encapsulation
