# Static Methods in Python (`@staticmethod`)

In Python, methods defined inside a class usually operate on a specific object instance. These methods receive a special parameter called `self`, which provides access to the object's attributes and methods.

However, sometimes you need a function that logically belongs to a class but does not need access to:

- Instance variables (`self`)
- Class variables (`cls`)
- Object state

For such situations, Python provides **Static Methods**.

A Static Method is a regular function stored inside a class namespace and marked with the `@staticmethod` decorator.

---

# 1. What is a Static Method?

A Static Method is a method that:

✅ Belongs to the class

✅ Does not receive `self`

✅ Does not receive `cls`

✅ Cannot access instance attributes directly

✅ Cannot access class attributes automatically

---

## Syntax

```python
class ClassName:

    @staticmethod
    def method_name(parameters):
        # logic
```

---

# Why Use Static Methods?

Imagine you're building a utility class called:

```python
MathUtils
```

and you need helper functions such as:

```python
Addition
Subtraction
Percentage Calculation
Area Calculation
```

These functions don't depend on any object's internal data.

Instead of creating objects unnecessarily, you can place them inside a class as static methods.

---

# Real-World Analogy

Think of a calculator.

A calculator object might have:

```python
brand
model
battery_level
```

These are instance-specific properties.

---

However:

```python
5 + 10
```

does not depend on:

```python
brand
model
battery_level
```

The addition operation is universal.

Therefore:

```python
calculate_sum()
```

should be a static method.

---

# 2. Standard Instance Method

Instance methods work with object data.

---

## Example

```python
class MathUtils:

    def __init__(self, baseline_num):

        self.baseline_num = baseline_num

    def add_to_baseline(self, value):

        self.baseline_num += value

        return self.baseline_num
```

---

## Usage

```python
obj = MathUtils(10)

print(
    obj.add_to_baseline(5)
)
```

### Output

```text
15
```

---

## Why Does It Need `self`?

Because it accesses:

```python
self.baseline_num
```

which belongs to a specific object.

---

# Visual Representation

```text
Object

baseline_num = 10

        ▲
        │
        │ Uses
        │
add_to_baseline()
```

---

# 3. Creating a Static Method

Static methods use:

```python
@staticmethod
```

---

## Example

```python
class MathUtils:

    @staticmethod
    def calculate_sum(a, b):

        return a + b
```

---

Notice:

```python
No self
No cls
```

---

# Visual Representation

```text
MathUtils

└── calculate_sum()

No object required
```

---

# 4. Calling Static Methods

Static methods can be called directly from the class.

---

## Example

```python
result = MathUtils.calculate_sum(
    7,
    2
)

print(result)
```

### Output

```text
9
```

---

This is the preferred approach.

# 5. Calling Through an Object

Although static methods belong to the class, Python also allows calling them through an object.

---

## Example

```python
obj = MathUtils(10)

print(
    obj.calculate_sum(
        20,
        30
    )
)
```

### Output

```text
50
```

---

Even though it works, Python is actually doing:

```python
MathUtils.calculate_sum(
    20,
    30
)
```

behind the scenes.

---

# Complete Example

```python
class MathUtils:

    def __init__(self, baseline_num):

        self.baseline_num = baseline_num

    def add_to_baseline(self, value):

        self.baseline_num += value

        return self.baseline_num

    @staticmethod
    def calculate_sum(a, b):

        return a + b


result = MathUtils.calculate_sum(
    7,
    2
)

print(result)


obj = MathUtils(5)

print(
    obj.calculate_sum(
        10,
        20
    )
)
```

---

### Output

```text
9
30
```

# 6. Instance Methods vs Static Methods

---

## Instance Method

```python
class Demo:

    def greet(self):

        print("Hello")
```

Requires:

```python
obj = Demo()

obj.greet()
```

---

Python internally converts:

```python
obj.greet()
```

into:

```python
Demo.greet(obj)
```

---

The object itself becomes:

```python
self
```

---

# Static Method

```python
class Demo:

    @staticmethod
    def greet():

        print("Hello")
```

---

Usage:

```python
Demo.greet()
```

No object is passed.

---

No automatic argument injection occurs.

---

# Internal Difference

---

## Instance Method

```python
obj.method()
```

Internally:

```python
Class.method(obj)
```

---

## Static Method

```python
Class.method()
```

Internally:

```python
Class.method()
```

Nothing extra is passed.

---

# 7. When Should You Use Static Methods?

Use a static method when:

---

## Rule 1

The method doesn't need:

```python
self
```

---

## Rule 2

The method doesn't modify:

```python
instance variables
```

---

## Rule 3

The method doesn't depend on:

```python
object state
```

---

# Good Examples

---

## Mathematical Utilities

```python
class MathUtils:

    @staticmethod
    def square(x):

        return x * x
```

---

## Temperature Conversion

```python
class Temperature:

    @staticmethod
    def celsius_to_fahrenheit(c):

        return (c * 9/5) + 32
```

---

## String Formatting

```python
class StringUtils:

    @staticmethod
    def capitalize_word(word):

        return word.capitalize()
```

---

# Bad Example

```python
class BankAccount:

    @staticmethod
    def deposit(amount):

        self.balance += amount
```

This fails because:

```python
self
```

does not exist inside a static method.

---

# 8. Benefits of Static Methods

---

## Better Organization

Instead of polluting global scope:

```python
square()
cube()
factorial()
```

you can organize them:

```python
MathUtils.square()
MathUtils.cube()
MathUtils.factorial()
```

---

## Improved Readability

The decorator immediately tells developers:

```text
This method does not modify object state.
```

---

## Reduced Overhead

Python avoids:

```python
self
```

injection.

Less work is performed during method calls.

---

## Cleaner Namespaces

Related functionality remains grouped together.

---

# 9. Static Methods vs Class Methods vs Instance Methods

Python provides three method types.

---

## Instance Method

```python
def method(self):
```

Access:

✅ Object variables

✅ Object methods

---

## Class Method

```python
@classmethod

def method(cls):
```

Access:

✅ Class variables

❌ Object variables

---

## Static Method

```python
@staticmethod

def method():
```

Access:

❌ Object variables

❌ Class variables (automatically)

---

# Quick Example

```python
class Demo:

    company = "Google"

    def instance_method(self):

        print("Instance")

    @classmethod
    def class_method(cls):

        print(cls.company)

    @staticmethod
    def static_method():

        print("Utility")
```

---

# Summary Table

| Feature | Instance Method | Class Method | Static Method |
|----------|----------------|-------------|---------------|
| Decorator Required | ❌ No | ✅ `@classmethod` | ✅ `@staticmethod` |
| Receives `self` | ✅ Yes | ❌ No | ❌ No |
| Receives `cls` | ❌ No | ✅ Yes | ❌ No |
| Access Instance Variables | ✅ Yes | ❌ No | ❌ No |
| Access Class Variables | ✅ Yes | ✅ Yes | ❌ Automatic Access |
| Requires Object | Usually Yes | No | No |
| Primary Purpose | Work with object state | Work with class state | Utility/helper logic |

---

# Key Takeaways

### Static Methods

Are regular functions stored inside a class.

---

### Decorator

```python
@staticmethod
```

---

### They Do Not Receive

```python
self
```

or

```python
cls
```

---

### Ideal For

✅ Utility functions

✅ Mathematical helpers

✅ String processing

✅ Data conversion

---

### Preferred Invocation

```python
ClassName.method()
```

Example:

```python
MathUtils.calculate_sum(
    10,
    20
)
```

---

### Most Important Rule

Use a static method whenever a function logically belongs to a class but does **not require access to object data or class data**.

This keeps your code organized, readable, and aligned with professional OOP design practices.
