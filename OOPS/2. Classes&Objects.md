# Classes and Objects in Python: Syntax and Practical Implementation

Building upon the conceptual foundation of Object-Oriented Programming (OOPs), this section focuses on the actual syntax used to create **Classes** and **Objects** in Python.

Classes act as blueprints, while objects are real instances created from those blueprints.

---

# 1. Defining a Class

A class is created using the `class` keyword.

By convention, class names follow **PascalCase** naming style.

### Syntax

```python
class ClassName:
    pass
```

---

## Example: Creating a Person Class

```python
class Person:

    # Class Attributes
    name = "Harry"
    occupation = "Software Developer"
    net_worth = 10

    # Instance Method
    def info(self):
        print(f"{self.name} is a {self.occupation}")
```

---

## Components Breakdown

### Class Name

```python
class Person:
```

Defines a blueprint called `Person`.

---

### Attributes

```python
name = "Harry"
occupation = "Software Developer"
net_worth = 10
```

These variables belong to the class and act as default values.

---

### Method

```python
def info(self):
```

Defines behavior that objects created from the class can perform.

---

# 2. Instantiating Objects (Creating Instances)

Once a class exists, you can create objects from it.

### Syntax

```python
object_name = ClassName()
```

---

## Example

```python
a = Person()
b = Person()
c = Person()
```

Three separate objects are now created.

---

### Memory Representation

```text
Person Class
      │
      ├────► a
      │
      ├────► b
      │
      └────► c
```

Each object occupies its own memory space.

---

# 3. Accessing Attributes

Attributes can be accessed using dot notation.

### Example

```python
a = Person()

print(a.name)
print(a.occupation)
```

### Output

```text
Harry
Software Developer
```

Since no custom values were assigned, Python uses the default class attributes.

---

# 4. Modifying Unique Object Properties

Each object can maintain its own independent data.

Changing one object does not affect others.

---

## Example

```python
a = Person()
b = Person()
c = Person()

a.name = "Shubham"
a.occupation = "Accountant"

b.name = "Nitika"
b.occupation = "HR Manager"
```

---

### Current Object State

```text
Object a
---------
Name       = Shubham
Occupation = Accountant

Object b
---------
Name       = Nitika
Occupation = HR Manager

Object c
---------
Name       = Harry
Occupation = Software Developer
```

Notice that object `c` still uses the default values.

---

# 5. Calling Methods

Methods are invoked using dot notation.

### Example

```python
a.info()
b.info()
c.info()
```

### Output

```text
Shubham is a Accountant
Nitika is a HR Manager
Harry is a Software Developer
```

---

# 6. Understanding the `self` Parameter

One of the most important concepts in Python OOP is the `self` parameter.

---

## Definition

`self` refers to the current object that is executing the method.

It allows Python to know:

```text
Which object's data should be accessed?
```

---

### Example

```python
class Person:

    name = "Harry"

    def show_name(self):
        print(self.name)
```

---

When:

```python
a = Person()

a.name = "Shubham"

a.show_name()
```

Output:

```text
Shubham
```

Python understands that:

```python
self
```

should refer to object `a`.

---

# 7. What Happens Internally?

Many beginners think:

```python
a.info()
```

is executed directly.

Internally, Python converts it into:

```python
Person.info(a)
```

---

### Example

When you write:

```python
a.info()
```

Python automatically does:

```python
Person.info(a)
```

The object itself is passed as the first argument.

Therefore:

```python
self == a
```

inside the method.

---

# Visual Explanation of `self`

Consider:

```python
a = Person()
b = Person()
```

### Call 1

```python
a.info()
```

Internally:

```python
Person.info(a)
```

```python
self → a
```

---

### Call 2

```python

```

Internally:

```python

```


Same method.

Different output.

# 8. Complete Example

```python
class Person:

    name = "Harry"
    occupation = "Software Developer"

    def info(self):
        print(f"{self.name} is a {self.occupation}")


a = Person()
b = Person()
c = Person()

a.name = "Shubham"
a.occupation = "Accountant"

b.name = "Nitika"
b.occupation = "HR Manager"

a.info()
b.info()
c.info()
```

### Output

```text
Shubham is a Accountant
Nitika is a HR Manager
Harry is a Software Developer
```

# 9. Class Attributes vs Instance Attributes

This example introduces an important distinction.

---

## Class Attribute

Defined directly inside the class.

```python
class Person:
    name = "Harry"
```

Shared as a default value.

---

## Instance Attribute

Defined specifically for one object.

```python
a.name = "Shubham"
```

Belongs only to object `a`.

---

### Example

```python
class Person:
    name = "Harry"

a = Person()
b = Person()

a.name = "Shubham"
```

---

### Result

```python
print(a.name)
```

Output:

```text
Shubham
```

```python
print(b.name)
```

Output:

```text
Harry
```

---

# Attribute Lookup Process

When Python encounters:

```python
a.name
```

it follows this search order:

### Step 1

Search inside object `a`.

```python
a.name ?
```

If found:

```text
Return Value
```

---

### Step 2

If not found, search inside class.

```python
Person.name ?
```

If found:

```text
Return Default Value
```

---

### Step 3

If still not found:

```python
AttributeError
```

is raised.

# Summary Table

| Concept | Description |
|----------|------------|
| Class | Blueprint/template for creating objects |
| Object | Real instance created from a class |
| Attribute | Variable belonging to a class or object |
| Method | Function defined inside a class |
| `self` | Reference to the current object |
| Class Attribute | Shared default value defined in class |
| Instance Attribute | Unique value stored inside an object |

---

# Key Rules to Remember

### Object Isolation

Changes to one object do not affect other objects.

```python
a.name = "Shubham"
```

does not change:

```python
b.name
```

---

### The `self` Rule

Every instance method must accept:

```python
self
```

as its first parameter.

```python
def info(self):
```

---

### Automatic Passing

Python automatically supplies the current object.

```python
a.info()
```

becomes:

```python
Person.info(a)
```

---

### Default Fallback

If an instance attribute does not exist:

```python
a.name
```

Python checks the class attribute.
